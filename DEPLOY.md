# Выкат на хостинг — чек-лист

## Важно: выбор платформы

Приложение использует **better-sqlite3** (файловая БД) и **долго живущий Node-процесс**.  
**Vercel и другие serverless-платформы не подходят** — нет постоянного диска и поддержки нативных модулей.

Подойдут:
- **Railway**, **Render**, **Fly.io** — указать Node, добавить persistent volume для БД
- **VPS** (DigitalOcean, Timeweb, Selectel и т.п.) — развернуть через `npm run build && npm run start`, папка `data/` для БД

---

## Перед выкатом

### 1. Сборка
```bash
npm run build
npm run start
```
Убедиться, что сборка проходит и на `localhost:3000` всё работает (лендинг, запись, слоты).

### 2. Переменные окружения
Создать на хостинге переменные (по необходимости), см. `.env.example`:
- **DATABASE_PATH** — путь к файлу БД на постоянном томе (на VPS можно не ставить, тогда будет `data/barbershop.db` в проекте).
- **TELEGRAM_BOT_TOKEN** — если нужны уведомления мастеру в Telegram (в БД у мастера должен быть `telegramChatId`).
- **SMS_API_ID** — если нужны SMS через SMS.ru вместо/вместе с Telegram.

### 3. База данных
- При первом запуске БД создаётся и заполняется seed’ом (услуги, мастера Марат/Владимир).
- На хостинге с временной файловой системой нужно монтировать **persistent volume** и задать **DATABASE_PATH** в этот том, иначе при перезапуске БД обнулится.

### 4. Контент
- **Footer**: телефон, адрес, ссылки 2ГИС/Яндекс — проверить актуальность в `Footer.tsx`.
- **Страница /privacy** — есть; при необходимости отредактировать текст.
- Фото мастеров в блоке «Наши мастера» — пока инициалы; при желании добавить картинки в `public/` и подставить в `MastersSection.tsx`.

### 5. Домен и HTTPS
- Настроить домен на хостинге и включить HTTPS (Let’s Encrypt и т.д.).

---

## После выката

1. Открыть сайт, пройти сценарий: услуга → мастер → дата/время → форма → отправка.
2. Проверить, что запись появилась (в админке или напрямую в БД, если есть доступ).
3. Если включены Telegram/SMS — проверить, что мастер получил уведомление.

## Тест уведомления на телефон (Telegram)

Чтобы при создании записи приходило уведомление вам в Telegram:

### 1. Создать бота и получить токен
- Откройте Telegram, найдите **@BotFather**.
- Отправьте `/newbot`, придумайте имя и username бота.
- BotFather пришлёт **токен** вида `123456789:ABCdefGHI...`. Скопируйте его.

### 2. Узнать свой chat_id
- Напишите вашему боту **любое сообщение** (например, «Привет»).
- В браузере откройте (подставьте свой токен):
  ```
  https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates
  ```
- В ответе найдите `"chat":{"id": 123456789}` — число `123456789` это ваш **chat_id**.

### 3. Записать токен в проект
В корне проекта создайте файл `.env` (или добавьте в существующий):
```
TELEGRAM_BOT_TOKEN=123456789:ABCdefGHI...
```

### 4. Привязать chat_id к мастеру
В терминале из корня проекта:
```bash
node scripts/set-master-telegram.mjs 1 ВАШ_CHAT_ID
```
Здесь `1` — id мастера Марат (для Владимира используйте `2`). Подставьте свой chat_id вместо `ВАШ_CHAT_ID`.

### 5. Перезапустить приложение
```bash
npm run dev
```
Сделайте тестовую запись на этого мастера — в Telegram должно прийти сообщение вида:  
«Новая запись: Имя, Услуга, 04.03.2026 14:00. Barbershop».

---

**SMS вместо Telegram:** если нужны уведомления по SMS, зарегистрируйтесь на [SMS.ru](https://sms.ru), получите `api_id`, добавьте в `.env` переменную `SMS_API_ID=...`. Уведомление уйдёт на номер телефона мастера из БД (для теста можно временно поменять номер мастера в БД на свой).
